# Contributing

Thanks for your interest in contributing! A few quick guidelines to get started:

- File issues for bugs or feature requests so we can discuss before large changes.
- Fork the repo and create a branch named `feature/your-short-desc` or `fix/issue-number`.
- Keep changes small and focused; include screenshots or short recordings for UI changes.
- Write clear commit messages and reference GitHub issue numbers when relevant.
- Run linters/formatters locally before opening a PR. If you add code, include tests where practical.
- Open a pull request and describe the intent, what changed, and how to test it.

Maintainers will review and request changes if necessary. Thanks! ❤️
